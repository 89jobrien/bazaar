# bazaar showcase ŌĆö Design Spec

## Goal

Turn the `bazaar` repo into a personal open-source showcase: a structured `README.md` overview
plus a self-contained `index.html` generated by a `bz` Rust binary, covering all public GitHub
repos, crates.io packages, PyPI packages, and Claude Code plugins ŌĆö refreshed automatically via
GitHub Actions.

---

## Scope

**In scope:**
- Public GitHub repos pushed in the last 6 months (any language)
- crates.io crates owned by the user
- PyPI packages listed in a `pypi.toml` config file (no user-listing API exists)
- Claude Code plugins listed in `.claude-plugin/marketplace.json`
- A generated `README.md` (concise, human-authored sections + generated project table)
- A generated `index.html` (single self-contained file, no JS framework, inline CSS)
- GitHub Actions workflow: runs on push + daily cron, commits updated output files

**Out of scope:**
- Authentication-gated data
- Interactive filtering/search
- Any JS framework or external CSS

---

## Binary: `bz`

Crate: `bazaar-gen` (produces binary named `bz`)

### Invocation

```
bz [--output index.html] [--readme README.md]
```

Defaults: writes `index.html` and `README.md` in the current directory.

### Configuration (no hardcoded PII)

All user-specific values come from environment variables or config files:

| Source | Variable / File | Purpose |
|---|---|---|
| env | `GITHUB_TOKEN` | GitHub API auth (5000 req/hr) |
| env | `GITHUB_USER` | GitHub login to fetch repos for |
| env | `CRATES_IO_USER` | crates.io login (may differ from GitHub) |
| file | `pypi.toml` | List of PyPI package names to include |

`pypi.toml` format:
```toml
packages = ["package-one", "package-two"]
```

If `GITHUB_TOKEN` is absent, the binary falls back to unauthenticated (60 req/hr) and prints a
warning. If `GITHUB_USER` or `CRATES_IO_USER` are absent, the binary exits with a clear error.

### Data Sources

**GitHub** (`https://api.github.com`)
- `GET /users/{GITHUB_USER}/repos?type=public&sort=pushed&per_page=100`
- Filter: `pushed_at` within last 6 months
- Fields: `name`, `description`, `html_url`, `language`, `pushed_at`, `stargazers_count`,
  `topics`, `homepage`
- Latest release: `GET /repos/{owner}/{repo}/releases/latest` (best-effort, 404 = no release)

**crates.io** (`https://crates.io/api/v1`)
- `GET /users/{CRATES_IO_USER}` ŌåÆ extract numeric `id`
- `GET /crates?user_id={id}&per_page=100` (paginate until exhausted)
- Fields: `name`, `description`, `updated_at`, `max_version`, `downloads`

**PyPI** (`https://pypi.org/pypi`)
- For each package in `pypi.toml`: `GET /pypi/{name}/json`
- Fields: `info.name`, `info.version`, `info.summary`, `info.home_page`

**Plugins** (local file)
- Read `.claude-plugin/marketplace.json` from the repo root
- Fields: `name`, `description`, `source.repo`

### Unified Data Model

```rust
struct Project {
    name: String,
    description: Option<String>,
    url: String,
    kind: Vec<Kind>,          // a repo can be Both GitHub + Crate
    language: Option<String>,
    pushed_at: Option<DateTime<Utc>>,
    version: Option<String>,
    stars: Option<u32>,
    downloads: Option<u64>,   // crates.io only
}

enum Kind {
    GitHubRepo,
    CratesIo,
    PyPI,
    ClaudePlugin,
}
```

Projects are merged by name (a GitHub repo that matches a crates.io crate name gets both badges).
Sorted by `pushed_at` descending, `None` last.

### Output: `index.html`

Single self-contained HTML file. No external assets. Inline CSS only (dark theme, monospace
aesthetic). Structure:

```
<header>  name | tagline | GitHub link
<nav>     All | Plugins | Crates | PyPI | Repos
<section> Cards grid ŌĆö one card per Project
<footer>  "Generated YYYY-MM-DD HH:MM UTC"
```

Each card shows: name (linked), badges for each Kind, description, language, version/release,
stars (if >0), last pushed date.

The nav filters are pure CSS (`:target` selector or checkbox hack) ŌĆö zero JavaScript.

### Output: `README.md`

Human-authored header section (pulled from a `README.header.md` file in the repo root, or a
default if absent), followed by a generated project table:

```markdown
| Project | Kind | Description | Updated |
|---|---|---|---|
| [name](url) | Plugin / Crate / Repo | description | YYYY-MM-DD |
```

Sorted same as HTML output.

## GitHub Actions Workflow

File: `.github/workflows/generate.yml`

Triggers:
- `push` to `main`
- `schedule`: daily at 06:00 UTC

Steps:
1. Checkout repo
2. Install Rust stable
3. `cargo build --release -p bazaar-gen`
4. Run `bz --output index.html --readme README.md`
   - Env: `GITHUB_TOKEN` (from secrets), `GITHUB_USER` (from vars), `CRATES_IO_USER` (from vars)
5. If files changed, commit and push (`[skip ci]` suffix to avoid loops)

GitHub Pages: serve from `main` branch root (`index.html`).

## Crate Structure

```
bazaar-gen/
  Cargo.toml         (binary name = "bz")
  src/
    main.rs          (CLI arg parsing, orchestration)
    config.rs        (env var + pypi.toml loading)
    fetch/
      mod.rs
      github.rs
      crates_io.rs
      pypi.rs
      plugins.rs
    model.rs         (Project, Kind, merge logic)
    render/
      mod.rs
      html.rs        (Askama template rendering)
      markdown.rs    (README table generation)
    template/
      index.html     (Askama template, include_str! embedded)
```

## Error Handling

- Missing required env vars ŌåÆ exit 1 with message naming the missing var
- API errors (4xx/5xx) ŌåÆ log warning, skip that source, continue
- PyPI 404 for a named package ŌåÆ log warning, skip
- Rate limit (403/429) ŌåÆ log error with retry-after if present, exit 1
- File write errors ŌåÆ exit 1

---

## Testing

- Unit tests for `model.rs` merge logic (GitHub repo + matching crate name ŌåÆ single Project
  with both Kind badges)
- Unit tests for `config.rs` (missing env var detection, pypi.toml parsing)
- Integration tests are skipped (require live API calls); the CI workflow itself serves as the
  integration test
